---
title: "MediaLibrary.sqlitedb Schema Reference for ByeTunes"
description: "Detailed schema reference for MediaLibrary.sqlitedb: the iOS native media database that ByeTunes reads and writes during music injection."
---

`MediaLibrary.sqlitedb` is the SQLite database that iOS uses internally to track everything in the Apple Music library. ByeTunes downloads this file from the device, merges new song records into it using `MediaLibraryBuilder`, and re-uploads it. This page documents the key tables and columns that ByeTunes reads and writes.

<Note>
  ByeTunes never modifies this database in place on the device. It always downloads it, modifies a local copy, and re-uploads the result. This ensures the database is consistent before iOS reloads it.
</Note>

## PRAGMA user_version

The `user_version` pragma identifies which schema revision is in use. ByeTunes writes the correct value based on the detected iOS version:

| iOS version | Schema family | `user_version` |
|---|---|---|
| iOS 16 | `ios16` | 2320030 |
| iOS 17, 18 | `ios17Or18` | 2120020 |
| iOS 26 | `ios26` | 2320030 |
| iOS 26 (subscription) | `ios26Subscription` | 2310000 |

## base_location

The `base_location` table maps integer IDs to filesystem path prefixes. ByeTunes ensures two rows always exist:

| `base_location_id` | `path` | Used for |
|---|---|---|
| 3840 | `iTunes_Control/Music/F00` | Music audio files |
| 3900 | `iTunes_Control/Ringtones` | Ringtone files |

The `item.base_location_id` column references this table. Every song injected by ByeTunes uses `base_location_id = 3840`.

## item

The core song record. One row per media item.

| Column | Type | Description |
|---|---|---|
| `item_pid` | INTEGER (PK) | Persistent ID — a random `Int64` generated by `SongMetadata.generatePersistentId()` |
| `media_type` | INTEGER | `8` = music |
| `base_location_id` | INTEGER | `3840` for music (resolves to `iTunes_Control/Music/F00`) |
| `item_artist_pid` | INTEGER | Foreign key into `item_artist` |
| `album_pid` | INTEGER | Foreign key into `album` |
| `album_artist_pid` | INTEGER | Foreign key into `album_artist` |
| `genre_id` | INTEGER | Foreign key into `genre` |
| `disc_number` | INTEGER | Disc number (defaults to 1) |
| `track_number` | INTEGER | Track number within the disc |
| `in_my_library` | INTEGER | `1` = in library |
| `keep_local` | INTEGER | `1` = keep audio file on device |
| `date_added` | INTEGER | Unix timestamp when the item was added |
| `date_downloaded` | INTEGER | Unix timestamp of download (same as `date_added` for injected songs) |
| `is_compilation` | INTEGER | `0` = not a compilation |
| `exclude_from_shuffle` | INTEGER | `0` = included in shuffle |

## item_extra

Extended per-song metadata. Joined to `item` on `item_pid`.

| Column | Type | Description |
|---|---|---|
| `item_pid` | INTEGER (PK) | Matches `item.item_pid` |
| `title` | TEXT | Song title |
| `sort_title` | TEXT | Title used for sort ordering |
| `location` | TEXT | Audio filename (e.g., `ABCD1234EFGH.mp3`) relative to `base_location_id` |
| `total_time_ms` | INTEGER | Duration in milliseconds |
| `year` | INTEGER | Release year |
| `file_size` | INTEGER | File size in bytes |
| `disc_count` | INTEGER | Total number of discs |
| `track_count` | INTEGER | Total number of tracks on the disc |
| `location_kind_id` | INTEGER | `42` for local files |
| `copyright` | TEXT | Copyright string extracted from the file's metadata |
| `media_kind` | INTEGER | `1` = audio |
| `content_rating` | INTEGER | Explicit rating (from `rtng` atom for M4A; 0 = clean, 4 = explicit) |

## item_playback

Audio format and codec details for each song.

| Column | Type | Description |
|---|---|---|
| `item_pid` | INTEGER (PK) | Matches `item.item_pid` |
| `audio_format` | INTEGER | Four-CC integer or `301` for MP3. Examples: `1633772320` = AAC, `1716281667` = FLAC, `1463899717` = WAV |
| `bit_rate` | INTEGER | Bit rate in bits per second (defaults to `320` if unavailable) |
| `codec_type` | INTEGER | Core Audio format ID for the codec |
| `codec_subtype` | INTEGER | Codec subtype (from the stream basic description) |
| `sample_rate` | REAL | Sample rate in Hz (defaults to `44100.0` if unavailable) |
| `duration` | INTEGER | Raw duration value (additional field; `total_time_ms` in `item_extra` is the primary duration) |
| `data_kind` | INTEGER | `0` = local file |

## item_store

Apple Music catalog identifiers and store metadata.

| Column | Type | Description |
|---|---|---|
| `item_pid` | INTEGER (PK) | Matches `item.item_pid` |
| `store_item_id` | INTEGER | `cnID` — the iTunes catalog item ID |
| `storefront_id` | INTEGER | `sfID` — the storefront identifier |
| `store_composer_id` | INTEGER | `cmID` — the composer's catalog ID |
| `store_genre_id` | INTEGER | `geID` — the genre's catalog ID |
| `store_playlist_id` | INTEGER | `plID` — the album's catalog ID |
| `artist_id` | INTEGER | `atID` — the artist's catalog ID |
| `store_flavor` | TEXT | Comma-separated audio traits, e.g. `atmos,spatial` |
| `subscription_store_item_id` | INTEGER | Non-zero for Dolby Atmos / spatial audio capable tracks |
| `is_mastered_for_itunes` | INTEGER | `1` if the file was mastered for iTunes / Apple Digital Master |
| `date_released` | INTEGER | Release date as Core Data seconds (seconds since 2001-01-01) |
| `is_subscription` | INTEGER | `1` when using the `ios26Subscription` schema family |
| `sync_id` | INTEGER | Random PID used internally for sync tracking |

## item_artist

One row per unique artist name.

| Column | Type | Description |
|---|---|---|
| `item_artist_pid` | INTEGER (PK) | Random persistent ID for the artist |
| `item_artist` | TEXT | Artist name as displayed |
| `sort_item_artist` | TEXT | Name used for sort ordering (same as `item_artist` by default) |
| `grouping_key` | BLOB | Binary sort key derived from the artist name (uppercase alphabetic bytes, `0x04` for space, `0x0A` for `/`) |
| `representative_item_pid` | INTEGER | `item_pid` of the first song added for this artist |
| `store_id` | INTEGER | Artist catalog ID (`atID`) |
| `sync_id` | INTEGER | Random PID for sync tracking |
| `keep_local` | INTEGER | `1` |

## album

One row per unique album name.

| Column | Type | Description |
|---|---|---|
| `album_pid` | INTEGER (PK) | Random persistent ID for the album |
| `album` | TEXT | Album name as displayed |
| `album_year` | INTEGER | Release year of the album |
| `album_artist_pid` | INTEGER | Foreign key into `album_artist` |
| `grouping_key` | BLOB | Binary sort key (same algorithm as `item_artist.grouping_key`) |
| `store_id` | INTEGER | Album catalog ID (`plID`) — present on iOS 17/18/26 schemas only |
| `representative_item_pid` | INTEGER | `item_pid` of the first song in the album |

## album_artist

One row per unique album artist. Separate from `item_artist` to support "Various Artists" and similar cases.

| Column | Type | Description |
|---|---|---|
| `album_artist_pid` | INTEGER (PK) | Random persistent ID |
| `album_artist` | TEXT | Album artist name |
| `sort_album_artist` | TEXT | Name used for sort ordering |
| `grouping_key` | BLOB | Binary sort key |
| `representative_item_pid` | INTEGER | `item_pid` of the first associated song |
| `sort_order` | INTEGER | Sort order index — present on iOS 17/18/26 schemas only |
| `sort_order_section` | INTEGER | Section letter index for the sort order |
| `store_id` | INTEGER | Artist catalog ID |

## genre

One row per unique genre string.

| Column | Type | Description |
|---|---|---|
| `genre_id` | INTEGER (PK) | Random persistent ID |
| `genre` | TEXT | Canonical genre string (unknown genres normalize to `"Music"`) |
| `representative_item_pid` | INTEGER | `item_pid` of the first song in this genre |

---

## artwork

One row per artwork file stored on the device.

| Column | Type | Description |
|---|---|---|
| `artwork_token` | TEXT (PK) | String token identifying this artwork; for injected songs this is the `item_pid` as a string |
| `artwork_source_type` | INTEGER | `1` = local file; `300` = iOS 26.4+ local source (additional row written alongside type `1`) |
| `relative_path` | TEXT | Path relative to `/iTunes_Control/iTunes/Artwork/Originals/`. Computed as the SHA1 of the token string: first 2 hex chars = subdirectory, remaining chars = filename |
| `artwork_type` | INTEGER | `1` = standard; `6` = iOS 26.4+ display variant |
| `interest_data` | TEXT | JSON object containing color analysis data (`backgroundColor`, `primaryTextColor`, etc.) |
| `artwork_variant_type` | INTEGER | `0` — present only on iOS 26 schemas |

<Tip>
  To locate an artwork file on the device manually, SHA1-hash the `artwork_token` string and split the result: the first two characters are the subdirectory, the rest is the filename under `/iTunes_Control/iTunes/Artwork/Originals/`.
</Tip>

## artwork_token

Links an artwork entry to a specific entity (song, album, or artist).

| Column | Type | Description |
|---|---|---|
| `artwork_token` | TEXT | References `artwork.artwork_token` |
| `entity_pid` | INTEGER | PID of the entity this artwork belongs to |
| `entity_type` | INTEGER | `0` = item (song), `1` = album, `2` = artist, `4` = album display |
| `artwork_source_type` | INTEGER | Matches `artwork.artwork_source_type` |
| `artwork_type` | INTEGER | Matches `artwork.artwork_type` |
| `artwork_variant_type` | INTEGER | `0` — iOS 26 schemas only |

ByeTunes inserts four `artwork_token` rows per song (entity types 0, 1, 2, and 4) so that the artwork appears on the track, album, and artist views.

---

## best_artwork_token

Stores the "winning" artwork token per entity for display purposes. iOS uses this table to quickly look up which artwork to render without scanning all candidates.

| Column | Type | Description |
|---|---|---|
| `entity_pid` | INTEGER | PID of the entity |
| `entity_type` | INTEGER | Same type codes as `artwork_token` |
| `artwork_type` | INTEGER | `1` = standard, `6` = iOS 26.4+ display |
| `available_artwork_token` | TEXT | The token of the artwork to display |
| `fetchable_artwork_token` | TEXT | Empty string for local artwork |
| `fetchable_artwork_source_type` | INTEGER | `0` for local artwork |
| `artwork_variant_type` | INTEGER | `0` — iOS 26 schemas only |

## lyrics

Plain-text lyrics per song.

| Column | Type | Description |
|---|---|---|
| `item_pid` | INTEGER (PK) | Matches `item.item_pid` |
| `lyrics` | TEXT | Cleaned plain-text lyrics (LRC timestamps and bracketed metadata stripped) |
| `store_lyrics_available` | INTEGER | `1` — signals to iOS that lyrics are present |
| `time_synced_lyrics_available` | INTEGER | `1` — signals that synced lyrics may be available |
| `downloaded_catalog_lyrics_available` | INTEGER | `0` — present only on iOS 26 schemas |

---

## sort_map

The `sort_map` table stores normalized sort keys (used for title, artist, album, and genre ordering). ByeTunes calls `insertSortMap` to upsert entries for every new string and receives an `(order, section)` pair that is written into the corresponding `_order` and `_order_section` columns on `item`.

## Relationships summary

```
item  ──── item_extra          (1:1, via item_pid)
      ──── item_playback        (1:1, via item_pid)
      ──── item_stats           (1:1, via item_pid)
      ──── item_store           (1:1, via item_pid)
      ──── lyrics               (1:1, via item_pid)
      ──── chapter              (1:1, via item_pid)
      ──┬─ item_artist          (N:1, via item_artist_pid)
        ├─ album                (N:1, via album_pid)
        └─ genre                (N:1, via genre_id)

album ──── album_artist         (N:1, via album_artist_pid)

artwork ── artwork_token        (1:N, via artwork_token)
        ── best_artwork_token   (1:N, via available_artwork_token)
```

For a higher-level view of how `MediaLibraryBuilder` populates these tables during injection, see the [Architecture overview](/build/architecture).
