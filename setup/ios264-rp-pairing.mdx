Starting with iOS 26.4, Apple introduced a new device authentication format called RP Pairing. The older lockdown pairing file that works on iOS 16.0 through 26.3.x is no longer accepted on these newer versions. ByeTunes detects your iOS version automatically and, when it sees iOS 26.4 or later, prompts you to import an RP Pairing File instead. The process for generating one uses the same `idevice_pair` tool as before, but produces a different output file format — `rpPairingFile.plist` — that is stored separately from the regular pairing file.

<Note>
  This page only applies if your iPhone is running **iOS 26.4 or newer**. If you are on iOS 16.0 through iOS 26.3.x, follow the [regular pairing file guide](/setup/pairing-file) instead.
</Note>

## What is an RP Pairing File?

An RP Pairing File is Apple's new authentication credential format introduced in iOS 26.4. It replaces the lockdown-based pairing flow for accessing device services. Where the old pairing file used a TCP connection through the lockdown daemon on the standard port, the RP Pairing File opens a heavier tunnel through Remote Service Discovery (RSD) and the AFC protocol on port `49152`. ByeTunes uses this tunnel for all device access on iOS 26.4 and later, including file uploads, database writes, and killing the Music app before injection.

## How ByeTunes detects the requirement

ByeTunes checks the iOS version at startup by reading the device's operating system version. If the major version is 26 and the minor version is 4 or higher (or if the major version exceeds 26), the app switches into RP Pairing mode. In this mode:

- The onboarding screen asks for an **RP Pairing File** rather than a regular pairing file.
- If you previously had a regular pairing file from an older iOS version, the **RPPairingUpgradePrompt** overlay appears automatically over the main interface and stays on screen until you import the correct RP Pairing File.
- Importing a regular pairing file when RP Pairing mode is active will fail validation with an error message — the files are not interchangeable.

## Generate and import the RP Pairing File

<Steps>
  <Step title="Install Rust and idevice_pair (if not already done)">
    If you have not set up `idevice_pair` yet, install Rust first:

    ```bash
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    ```

    Then install `idevice_pair`:

    ```bash
    cargo install --git https://github.com/jkcoxson/idevice_pair
    ```

    See the [idevice_pair repository](https://github.com/jkcoxson/idevice_pair) for full details.
  </Step>

  <Step title="Connect your iPhone and run idevice_pair for RP Pairing">
    Connect your iPhone via USB or ensure it is on the same Wi-Fi network as your computer. Run:

    ```bash
    idevice_pair rp-pair
    ```

    iOS will prompt you to **Trust This Computer** if you have not done so before. Tap **Trust** and enter your passcode. `idevice_pair` then generates the RP Pairing File and prints its location.

    <Tip>
      If the command is not recognized, make sure you are on a recent version of `idevice_pair` that includes RP Pairing support. Run `cargo install --git https://github.com/jkcoxson/idevice_pair` again to update.
    </Tip>
  </Step>

  <Step title="Transfer the RP Pairing File to your iPhone">
    Locate the `.plist` file printed by `idevice_pair` and send it to your iPhone:

    <Tabs>
      <Tab title="AirDrop">
        Right-click the file in Finder or File Explorer and choose **Share → AirDrop**. Select your iPhone. Tap **Accept** on your iPhone and save the file to the **Files** app.
      </Tab>
      <Tab title="Files app or email">
        Email the file to yourself and open the attachment on your iPhone, or upload it to iCloud Drive and access it through the **Files** app.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Import the file through the RP Pairing upgrade prompt">
    Open **ByeTunes**. If you are on iOS 26.4 or newer and have not yet imported an RP Pairing File, the **RPPairingUpgradePrompt** overlay appears automatically. Tap **Import RP Pairing File** and navigate to the `.plist` file you just transferred.

    ByeTunes validates the file. If it is a valid RP Pairing File, the prompt dismisses and the app begins connecting over the RP tunnel. If validation fails, an error message is shown in the prompt — confirm you selected the file generated by `idevice_pair rp-pair` and not a regular pairing file, then try again.

    <Warning>
      The **RPPairingUpgradePrompt** overlay will remain on screen and block access to the rest of the app until a valid RP Pairing File is imported. It cannot be dismissed without supplying the correct file.
    </Warning>
  </Step>
</Steps>

## Performance note for iOS 26.4+

Injection on iOS 26.4 and newer is slightly slower than on older iOS versions. This is because the RP Pairing method routes all communication through a heavier tunnel/RSD/AFC path instead of the direct lockdown TCP connection used by regular pairing files. ByeTunes reduces extra round trips where possible — for example, it reuses the AFC connection across the full upload pass rather than reconnecting for each file — but the underlying Apple-required method adds overhead that cannot be avoided.

<Info>
  If injection appears to stall, it is most likely still working. The RP tunnel setup itself takes a moment before file transfers begin. Avoid closing ByeTunes or disconnecting the VPN during this time.
</Info>

## Where the RP Pairing File is stored

After a successful import, ByeTunes saves the file as `rpPairingFile.plist` in the app group container, alongside (but separate from) the regular pairing file:

```
<App Group Container>/pairing file/rpPairingFile.plist
```

Both files can coexist — if you switch between devices running different iOS versions, ByeTunes will use whichever file is appropriate for the detected iOS version.
