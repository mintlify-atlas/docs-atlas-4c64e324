---
title: 'Generate and import a pairing file (iOS 16–26.3.x)'
description: 'Use idevice_pair to create a lockdown pairing file on your computer, then import it into ByeTunes on iOS 16.0 through iOS 26.3.x.'
---

Before ByeTunes can inject music into your Apple Music library, it needs a pairing file that proves it is authorized to communicate with your device. On iOS 16.0 through 26.3.x, this is a standard lockdown pairing file — a `.plist` file generated on your computer using the `idevice_pair` tool. Once generated, you transfer it to your iPhone and import it through the ByeTunes onboarding screen. You only need to do this once, and the file stays saved in the app.

<Note>
  If you are running **iOS 26.4 or newer**, you need an RP Pairing File instead. See [RP Pairing for iOS 26.4+](/setup/ios264-rp-pairing).
</Note>

## Generate and import the pairing file

<Steps>
  <Step title="Install Rust on your computer">
    `idevice_pair` is a Rust-based tool, so you need Rust installed before you can build it. Run the following command in your terminal:

    ```bash
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    ```

    Follow the on-screen prompts. After installation, close and reopen your terminal (or run `source ~/.cargo/env`) so the `cargo` command is available.
  </Step>

  <Step title="Install idevice_pair">
    Clone and install `idevice_pair` from its GitHub repository:

    ```bash
    cargo install --git https://github.com/jkcoxson/idevice_pair
    ```

    Refer to the [idevice_pair repository](https://github.com/jkcoxson/idevice_pair) for any platform-specific prerequisites or build instructions.
  </Step>

  <Step title="Connect your iPhone and run idevice_pair">
    Connect your iPhone to your computer via USB **or** ensure both your iPhone and computer are on the same Wi-Fi network. Then run:

    ```bash
    idevice_pair pair
    ```

    The first time you run this, iOS will show a **Trust This Computer?** prompt on your iPhone. Tap **Trust** and enter your passcode. Once trusted, `idevice_pair` generates the pairing file and prints the output path.

    <Tip>
      If `idevice_pair` cannot find your device, try running it over USB first to establish trust, then re-run over Wi-Fi.
    </Tip>
  </Step>

  <Step title="Locate the generated .plist file">
    After a successful pairing, `idevice_pair` saves a `.plist` file to your computer. The exact path is printed in the terminal output. Note this path — you need it in the next step.
  </Step>

  <Step title="Transfer the .plist file to your iPhone">
    Send the `.plist` file to your iPhone using one of these methods:

    <Tabs>
      <Tab title="AirDrop">
        In Finder or File Explorer, right-click the `.plist` file and choose **Share → AirDrop**. Select your iPhone from the AirDrop list. On your iPhone, tap **Accept** when the transfer prompt appears and save the file to the **Files** app.
      </Tab>
      <Tab title="Files app or email">
        Email the `.plist` file to yourself and open the attachment on your iPhone, or upload it to iCloud Drive and access it from the **Files** app. Any method that lands the file in the Files app works.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Import the pairing file into ByeTunes">
    Open **ByeTunes**. On first launch you will see the onboarding screen with an **Import Pairing File** button. Tap it and navigate to the `.plist` file you just transferred.

    ByeTunes automatically validates the file format when you select it. If the file is accepted, the app moves past the onboarding screen and begins connecting to your device. If validation fails, see the section below.
  </Step>
</Steps>

## What happens if you import the wrong file

ByeTunes validates every imported file before saving it. If you select a file that is not a valid lockdown pairing file — for example, if you accidentally pick an RP Pairing File on an older iOS version, or import an unrelated `.plist` — the app rejects it immediately with the error:

> That file is not a valid Pairing File.

The incorrect file is discarded and the import prompt stays open. Locate the correct `.plist` generated by `idevice_pair` and try again.

## Where the pairing file is stored

Once successfully imported, ByeTunes saves the file as `pairingFile.plist` inside the app group container at:

```
<App Group Container>/pairing file/pairingFile.plist
```

You do not need to re-import the file after app updates. If you ever need to regenerate the pairing (for example, after restoring your iPhone), just run `idevice_pair pair` again and repeat the import steps above.
